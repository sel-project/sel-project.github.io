sul.protocol.pocket132.play

Packets related to the gameplay. Network-related packets (encapsulation, acks, nacks) are managed by RakNet and every packet in this section is encapsualted in an Encapsualted packet.

  • Login

    Declaration

    class Login: sul.utils.buffer.Buffer;

    First MCPE packet sent after the establishment of the connection through raknet. It contains informations about the player.

    • protocol

      Declaration

      uint protocol;

      Version of the protocol used by the player.

    • body_

      Declaration

      sul.protocol.pocket132.types.LoginBody body_;

      Payload that contains 2 JWTs (with each length indicated by an unsigned little-endian 32-bits integer) with more informations about the player and its account.

  • PlayStatus

    Declaration

    class PlayStatus: sul.utils.buffer.Buffer;

    Packet sent as response to Login to indicate whether the connection has been accepted and when the player is ready to spawn in the world.

  • Disconnect

    Declaration

    class Disconnect: sul.utils.buffer.Buffer;

    Disconnects the player from the server.

    • hideDisconnectionScreen

      Declaration

      bool hideDisconnectionScreen;

      Indicates whether to display the main menu screen or a disconnection message.

    • message

      Declaration

      string message;

      The message to display in the disconnection screen. If the message is in the game's language file it will be translated client-side.

  • Text

    Declaration

    class Text: sul.utils.buffer.Buffer;

    Sends or receives a message from the player. Every variant's field can contain Minecraft's formatting codes.

    • Raw

      Declaration

      class Raw;

      Raw message that will be printed in the chat as it is.

    • Chat

      Declaration

      class Chat;

      Chat message sent by the player to the server. If sent from the server it will display as &lt;sender&gt; message.

      • sender

        Declaration

        string sender;

        Case sensitive name of the player that has sent the message.

      • message

        Declaration

        string message;

        Message sent by the player. It should be unformatted (regular expression: ยง[a-fA-F0-9k-or]) before being processed as chat message by the server.

    • Translation

      Declaration

      class Translation;

      Sends a message that will be translated client-side using the player's language.

      • message

        Declaration

        string message;

        A message in the game's language file.

      • parameters

        Declaration

        string[] parameters;

        Parameters that will be placed instead of the replacement symbols (%1, %2, etc...).

    • Popup

      Declaration

      class Popup;

      Displays popups messages for one tick before fading out. The popup messages are displayed at the centre of the screen and are not automatically aligned horizontally.

    • Tip

      Declaration

      class Tip;

      Displays a tip message for one tick before fading out. The tip message is displayed on top of the inventory bar and can contain multiple lines separated with \n.

    • Whisper

      Declaration

      class Whisper;

      Sends a whisper message to the client that will be displayed in the format `sender has whispered to you: message`.

  • SetTime

    Declaration

    class SetTime: sul.utils.buffer.Buffer;

    Sets the time.

    • time

      Declaration

      int time;

      Time of the day in a range from 0 to 24000. If higher or lower it will be moduled to 24000.

  • AddPlayer

    Declaration

    class AddPlayer: sul.utils.buffer.Buffer;

    Spawns a player after adding it to the player's list using PlayerList. If PlayerList is sent after this packet the player will appear to have the same skin as the player who receives this packet. Spawning a player to itself (using the same entity id given in the StartGame packet) will crash the client's game.

    • uuid

      Declaration

      sul.protocol.pocket132.types.McpeUuid uuid;

      Player's UUID, should match an UUID of a player in the list added through PlayerList.

    • username

      Declaration

      string username;

      Player's username and text displayed on the nametag if something else is not specified in the metadata.

  • RemoveEntity

    Declaration

    class RemoveEntity: sul.utils.buffer.Buffer;

    Despawns an entity or a player.

  • AddItemEntity

    Declaration

    class AddItemEntity: sul.utils.buffer.Buffer;

    Spawns a dropped item.

  • TakeItemEntity

    Declaration

    class TakeItemEntity: sul.utils.buffer.Buffer;

    Plays the collection animation and despawns the entity that has been collected.

    • collected

      Declaration

      long collected;

      Collected entity, usually an item entity or an arrow, that will float toward the collector and despawn.

    • collector

      Declaration

      long collector;

      Entity that collects, usually a player or another entity with an inventory.

  • AddPainting

    Declaration

    class AddPainting: sul.utils.buffer.Buffer;

    Spawns a painting entity in the world.

  • LevelSoundEvent

    Declaration

    class LevelSoundEvent: sul.utils.buffer.Buffer;

    Plays a sound at a certain position.

    • position

      Declaration

      Tuple!(float, "x", float, "y", float, "z") position;

      Position where the sound was generated. The closer to the player the more intense will be on the client.

  • UpdateAttributes

    Declaration

    class UpdateAttributes: sul.utils.buffer.Buffer;

    Updates an entity's attributes. This packet should be used when a value must be modified but it cannot be done using another packet (for example controlling the player's experience and level).

  • MobEquipment

    Declaration

    class MobEquipment: sul.utils.buffer.Buffer;

    Sent when the client puts an item in its hotbar or selects a new hotbar slot.

    • inventorySlot

      Declaration

      ubyte inventorySlot;

      Slot of the inventory where the item is. The hotbat slots (0-8) are not counted. 255 means that a generic empty slot has been selected.

    • hotbarSlot

      Declaration

      ubyte hotbarSlot;

      Slot of the hotbar where the item is being moved.

  • EntityFall

    Declaration

    class EntityFall: sul.utils.buffer.Buffer;

    Sent by the player when it falls from a distance that causes damage, that can be influenced by its armour and its effects.

    • distance

      Declaration

      float distance;

      Number of blocks the player has been in free falling before hitting the ground.

  • SetEntityData

    Declaration

    class SetEntityData: sul.utils.buffer.Buffer;

    Updates an entity's metadata.

  • SetEntityMotion

    Declaration

    class SetEntityMotion: sul.utils.buffer.Buffer;

    Updates an entity's motion.

    • entityId

      Declaration

      long entityId;

      Entity which motion is updated. If the entity id is the player's, its motion is updated client-side and the player will send movement packets to the server (meaning that the server has no physical calculations to do). If not an animation will be done client-side but the server will have to calculate the new position applying the item's movement rules.

    • motion

      Declaration

      Tuple!(float, "x", float, "y", float, "z") motion;

      New motion for the entity that will influence its movement.

  • AdventureSettings

    Declaration

    class AdventureSettings: sul.utils.buffer.Buffer;

    Updates the world's settings and client's permissions.

  • BlockEntityData

    Declaration

    class BlockEntityData: sul.utils.buffer.Buffer;

    Sets a block entity's nbt tag, block's additional data that cannot be indicated in the block's meta. More informations about block entities and their tag format can be found on Minecraft Wiki. The client sends this packet when it writes a sign.

    • position

      Declaration

      sul.protocol.pocket132.types.BlockPosition position;

      Position of the block that will be associated with tag.

    • nbt

      Declaration

      ubyte[] nbt;

      Named binary tag of the block. The format varies from the classic format of Minecraft: Pocket Edition (which is like Minecraft's but little endian) introducing the use of varints for some types: + The tag Int is encoded as a signed varint instead of a simple signed 32-bits integer + The length of the ByteArray and the IntArray is encoded as an unsigned varint instead of a 32-bits integer + The length of the String tag and the named tag's name length are encoded as an unisgned varint instead of a 16-bits integer

  • FullChunkData

    Declaration

    class FullChunkData: sul.utils.buffer.Buffer;

    Sends a 16x16 chunk to the client with its blocks, lights and block entities (tiles).

    • position

      Declaration

      Tuple!(int, "x", int, "z") position;

      Coordinates of the chunk.

  • SetCommandsEnabled

    Declaration

    class SetCommandsEnabled: sul.utils.buffer.Buffer;

    Indicates whether the cheats are enabled. If not the client cannot send commands.

  • SetDifficulty

    Declaration

    class SetDifficulty: sul.utils.buffer.Buffer;

    Sets the world's difficulty.

  • SetPlayerGameType

    Declaration

    class SetPlayerGameType: sul.utils.buffer.Buffer;

    Sets the player's gamemode. This packet is sent by the player when it has the operator status (set in AdventureSettings.permissions) and it changes the gamemode in the settings screen.

  • PlayerList

    Declaration

    class PlayerList: sul.utils.buffer.Buffer;

    Adds or removes a player from the player's list displayed in the pause menu. This packet should be sent before spawning a player with AddPlayer, otherwise the skin is not applied.

  • RequestChunkRadius

    Declaration

    class RequestChunkRadius: sul.utils.buffer.Buffer;

    Packet sent by the client when its view-distance is updated and when it spawns for the first time a world. A ChunkRadiusUpdate should always be sent in response, otherwise the player will not update its view distance.

    • radius

      Declaration

      int radius;

      Number of chunks before the fog starts to appear in the client's view. The value of this field is usually between 8 and 14.

  • ChunkRadiusUpdated

    Declaration

    class ChunkRadiusUpdated: sul.utils.buffer.Buffer;

    Packet sent always and only in response to RequestChunkRadius to change the client's view distance.

    • radius

      Declaration

      int radius;

      View distance that may be different from the client's one if the server sets a limit on the view distance.

  • GameRulesChanged

    Declaration

    class GameRulesChanged: sul.utils.buffer.Buffer;

    Updates client's game rules. This packet is ignored if the game is not launched as Education Edition and should be avoid in favour of AdventureSettings, with which the same result can be obtained with less data.

  • BossEvent

    Declaration

    class BossEvent: sul.utils.buffer.Buffer;

    Adds, removes or modifies an entity's boss bar. The percentage of the bar is calculated using the entity's attributes for the health and the max health, updated with UpdateAttributes.

  • AvailableCommands

    Declaration

    class AvailableCommands: sul.utils.buffer.Buffer;

    Sends a list of the commands that the player can use through the CommandStep packet.

  • Transfer

    Declaration

    class Transfer: sul.utils.buffer.Buffer;

    Transfers the player to another server. Once transferred the player will immediately close the connection with the transferring server, try to resolve the ip and join the new server starting a new raknet session.

    • ip

      Declaration

      string ip;

      Address of the new server. It can be an dotted ip (for example 127.0.0.1) or an URI (for example localhost or play.example.com). Only IP of version 4 are currently allowed.

    • port

      Declaration

      ushort port;

      Port of the new server. If 0 the server will try to connect to the default port.

  • SetTitle

    Declaration

    class SetTitle: sul.utils.buffer.Buffer;

    Displays titles on the client's screen.

    • text

      Declaration

      string text;

      Text that will be displayed in the place specified in the action field.
