    Properties

    NameTypeDescription
    alwaysConsumeable[get] boolIndicates whether the item can be consumed when the holder's hunger is full.
    attack[get] uintIndicates the damage caused by the item when used as a weapon. The value indicates the base damage without the influence of enchantments or effects.
    bedrock[get] boolIndicates whether the item exists in Minecraft.
    consumeable[get] boolIndicates whether or not an item can be eaten/drunk. If true, Item::onConsumed(Human consumer) will be called when this item is eaten/drunk.
    customName[get] stringGets the item's custom name.
    customName[set] stringSets the item's custom name.
    data[get] sul.items.ItemGets the item's data.
    finished[get] boolIf the item is a tool, checks whether its damage is higher than its durability.
    java[get] boolIndicates wether the item exists in Minecraft.
    javaCompound[get] sel.nbt.tags.CompoundGets the item's compound tag with the custom data of the item. It may be null if the item has no custom behaviours.
    javaId[get] ushortIndicates wether the item exists in Minecraft.
    javaMeta[get] ushortIndicates wether the item exists in Minecraft.
    lore[get] stringGets the item's lore or description which is displayed under the item's name when the item is hovered in the player's inventory.
    max[get] ubyteIndicates the highest number of items that can be stacked in a slot. This number is the default slot's count if not specified when created.
    name[get] stringGets the name (not the custom name!) of the item.
    placeable[get] boolIndicates whether or not the item can be placed. If this function returns true, Item::place(World world) will be probably called next for place a block
    pocketCompound[get] sel.nbt.tags.CompoundGets the item's compound tag with the custom data of the item. It may be null if the item has no custom behaviours.
    slot[get] SlotCreate a slot with the Item::max as count
    tool[get] boolIndicates whether the item is a tool. A tool can be used on blocks and entities and its meta will vary.
    toolMaterial[get] ubyteGets the tool's material if the item is a tool. Items with ID 0 have unspecified material, 1 is the minimum (wood) and 5 is the maximum (diamond).
    toolType[get] ubyteGets the item's tool type.
    unbreakable[get] boolIf the item is a tool, indicates whether the item is consumed when used for breaking or combat.

    Methods

    NameDescription
    addEnchantment Adds an enchantment to the item.
    attackOnEntity Function called when the item is used against an entity as a left click or screen tap.
    clear Removes the custom behaviours of the item, like custom name and enchantments.
    destroyOn Function called when the item is used to the destroy a block.
    onConsumed If consumeable is true, this function is called. when the item is eaten/drunk by its holder, who's passed as the first arguments.
    onPlaced Function called when the item is ready to be placed by a player (the event for the player has already been called).
    onReleased Function called when the item is released, usually after it has been throwed (which is used as aim-start function).
    onThrowed Function called when the item is throwed or aimed.
    opBinaryRight Adds an enchantment to the item.
    opBinaryRight Gets a pointer to the enchantment. This method can be used to check if the item has an enchantment and its level.
    opBinaryRight Removes an enchantment from the item.
    opEquals Deep comparation of 2 instantiated items. Compare ids, metas, custom names and enchantments.
    opEquals Compare an item with its type as a string or a group of strings.
    parseBedrockCompound Parses a compound, usually received from the client or saved in a world. The tag should never be null as the method doesn't check it.
    parseJavaCompound Parses a compound, usually received from the client or saved in a world. The tag should never be null as the method doesn't check it.
    place If Item::placeable returns true, this function should return an instance of the block that will be placed.
    removeEnchantment Removes an enchantment from the item.
    toString Returns the item as string in format "name" or "name:damage" for tools.
    useOnBlock Function called when the item is used on a block clicking the right mouse button or performing a long pressure on the screen.
    useOnEntity Function called when the item is used on an entity as right click or long screen pressure.
