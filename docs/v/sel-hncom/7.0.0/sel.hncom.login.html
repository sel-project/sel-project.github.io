sel.hncom.login

  • ConnectionRequest

    Declaration

    struct ConnectionRequest;

    First packet sent by the client after the connection is established. It contains informations used by the hub to check permissions and compatibility.

    • password

      Declaration

      string password;

      Password, if the hub requires one, or an empty string.

    • name

      Declaration

      string name;

      Name of the node that will be validated by the hub. It should always be lowercase and only contain letters, numbers, dashes and underscores.

    • main

      Declaration

      bool main;

      Indicates whether the node accepts clients when they first connect to the hub or exclusively when they are transferred.

    • protocol

      Declaration

      uint protocol;

      Version of the protocol used by the client that must match the hub's one.

  • ConnectionResponse

    Declaration

    struct ConnectionResponse;

    Reply to ConnectionRequest sent only when the node's ip is accepted by the hub. It contains the connection status (accepted or an error code) and the hub's protocol.

    • OUTDATED_HUB

      Declaration

      OUTDATED_HUB

      The hub uses an old version of hncom

    • OUTDATED_NODE

      Declaration

      OUTDATED_NODE

      The node uses an old version of hncom

    • PASSWORD_REQUIRED

      Declaration

      PASSWORD_REQUIRED

      A password is required to connect

    • WRONG_PASSWORD

      Declaration

      WRONG_PASSWORD

      The password doesn't match the hub's one

    • INVALID_NAME_LENGTH

      Declaration

      INVALID_NAME_LENGTH

      The name is too short or too long

    • INVALID_NAME_CHARACTERS

      Declaration

      INVALID_NAME_CHARACTERS

      The name contains invalid characters

    • NAME_ALREADY_USED

      Declaration

      NAME_ALREADY_USED

      There's already a node connected with the same name

    • NAME_RESERVED

      Declaration

      NAME_RESERVED

      The name cannot be used because the hub has reserved it for something else

    • BLOCKED_BY_PLUGIN

      Declaration

      BLOCKED_BY_PLUGIN

      A plugin has blocked the node from connecting

    • status

      Declaration

      ubyte status;

      Indicates the status of connection. If not 0, it indicates an error.

    • protocol

      Declaration

      uint protocol;

      Indicates the version of the protocol used by the hub when the status code indicates that the hub or the node is obsolete.

  • HubInfo

    Declaration

    struct HubInfo;

    Hub's informations.

    • serverId

      Declaration

      ulong serverId;

      Server's id, either given by a snoop system or randomly generated at runtime.

    • reservedUUIDs

      Declaration

      ulong reservedUUIDs;

      First number of the 4,294,967,296 (2^32) reserved by the hub to create the node's UUIDs. Every UUID generated by the node is formed by the server's id (most signicant) and the next reserved uuid (least significant). This way every UUID in the hub and in the connected nodes is always different.

    • displayName

      Declaration

      string displayName;

      Unformatted name of the server as indicated in the hub's configuration file.

    • gamesInfo

      Declaration

      GameInfo[ubyte] gamesInfo;

      Informations about the games supported by the hub.

    • online

      Declaration

      uint online;

      Number of players currently online and connected to other nodes.

    • max

      Declaration

      int max;

      Number of maximum players that can connect to the server (that is the sum of the max players of the nodes already connected). The number may change after the current node connects.

    • language

      Declaration

      string language;

      Default server's language in format (language_COUNTRY, e.g. en_GB) and also the default language for players that don't specify their language or for the ones which language is not supported by the server.

    • acceptedLanguages

      Declaration

      string[] acceptedLanguages;

      Languages accepted by the server in the same format as language. The list should always contain at least one element (the default language).

    • webAdmin

      Declaration

      bool webAdmin;

      Indicates whether the web admin protocol is active on the hub. If it is the node should send the port where it will listen for connections in its info packet.

    • additionalJSON

      Declaration

      JSONValue additionalJSON;

      Optional informations about the server's software, social accounts, system and options.

      Example:

      1. { "software": { "name": "selery", "version": "0.0.1", "stable": true }, "minecraft": { "edu": false, "realm": true }, "social": { "website": "example.com", "facebook": "example-official", "twitter": "example_tweets", "youtube": "examplechannel", "instagram": "example", "google-plus": "example-plus" }, "system": { "os": "Ubuntu 16.04", "cpu": "Intel(R) Core(TM) i5-5200U CPU @ 2.20GHz", "cores": 2, "ram": 2147483648 } }

  • NodeInfo

    Declaration

    struct NodeInfo;

    Node's informations.

    • acceptedGames

      Declaration

      uint[][ubyte] acceptedGames;

      Informations about the games accepted by the node. There should be at least one combination of game/protocols that is also accepted by hub as indicated in HubInfo.gamesInfo, otherwise the node will never receive any player.

    • max

      Declaration

      uint max;

      Maximum number of players accepted by node.

    • plugins

      Declaration

      Plugin[] plugins;

      List of plugins loaded on the node for creating queries on the hub.

    • webAdminPort

      Declaration

      ushort webAdminPort;

      Port where the node is listening for connections, if the web admin protocol is active on the hub.

    • additionalJSON

      Declaration

      JSONValue additionalJSON;

      Optional informations about the server's software and system, similar to HubInfo's additionalJson field.

      Example:

      1. { "software": { "name": "selery", "version": "0.0.1", "stable": true }, "system": { "os": "Windows 10", "cpu": "Intel(R) Core(TM) i7-5700U CPU @ 3.40GHz", "cores": 4, "ram": 8589934592 } }
