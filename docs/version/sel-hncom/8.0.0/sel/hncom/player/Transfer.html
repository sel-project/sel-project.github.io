    Struct Transfer

    Transfers a player to another node. When a player is transferred from the node the hub will not send the Remove packet and there's no way, for the node, to know whether the player was disconnected or successfully transferred, if not using messages through a user-defined protocol.

    struct Transfer ;

    Fields

    NameTypeDescription
    hubId uintPlayer to be transferred.
    message ubyte[]Optional data that will be always sent with the Add packet when the player is transferred. The content depends on the node's implementation or even by one of its plugins.
    node uintId of the node that player will be transferred to. It should be an id of a connected node (which can be calculated using AddNode and RemoveNode packets), otherwise the player will be disconnected or moved to another node (see the following field).
    onFail ubyteIndicates the action to be taken when a transfer fails because the indicated node is not connected anymore or it cannot accept the given player's game type or protocol. If the indicated node is full the player will be simply disconnected with the Server Full message.
