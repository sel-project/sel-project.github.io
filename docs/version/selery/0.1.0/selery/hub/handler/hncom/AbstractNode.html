    Class AbstractNode

    Session of a node. It's executed in a dedicated thread.

    class AbstractNode
      : sel.hncom.handler.Handler!(serverbound) ;

    Properties

    NameTypeDescription
    cpu[get] floatGets the node's usage, updated with the ResourcesUsage packet.
    full[get] boolIndicates whether the node is full.
    latency[get] uintGets the node's latency (it may not be precise).
    main[get] boolIndicates whether or not this is a main node. A main node is able to receive players without the use of the transfer function. Every hub should have at least one main node, otherwise every player that tries to connect will be disconnected with the 'end of stream' message.
    max[get] uintGets the highest number of players that can connect to the node.
    name[get] stringGets the name of the node. The name is different for every node connected to hub and it should be used other nodes with the transfer function.
    online[get] uintGets the number of players connected to the node.
    ram[get] ulongGets the node's usage, updated with the ResourcesUsage packet.
    tps[get] floatGets the node's usage, updated with the ResourcesUsage packet.
    worlds[get] shared(selery.hub.player.World)[]Gets the list of worlds loaded on the node.

    Methods

    NameDescription
    addNode Notifies the node that another node has connected to the hub.
    addPlayer Adds a player to the node.
    onClosed Called when the client closes the connection. Tries to transfer every connected player to the main node.
    onPlayerKicked Called when a player is kicked (not by the node).
    onPlayerLeft Called when a player lefts the server using the disconnect button or closing the socket.
    onPlayerTimedOut Called when a player times out.
    onPlayerTransferred Called when a player is transferred by the hub (not by the node) to another node.
    remoteCommand Executes a remote command.
    removeNode Notifies the node that another node has been disconnected from the hub.
    sendLatencyUpdate Updates a player's latency (usually sent every 30 seconds).
    sendMessage Sends a message to the node.
    sendPacketLossUpdate Updates a player's packet loss (usually sent every 30 seconds).
    sendTo Sends data to the node received from a player.
    updatePlayers Sends the number of online players and maximum number of players to the node.
    onPlayerGone Generic function that removes a player from the node's list and sends a PlayerDisconnected packet to notify the node of the disconnection.
