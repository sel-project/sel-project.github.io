sel.hncom.player

Packets related to a player. The first field of every packet is an hub id that uniquely identifies a player in the hub and never changes until it disconnects.

  • Add

    Declaration

    struct Add;

    Adds a player to the node.

    • FIRST_JOIN

      Declaration

      FIRST_JOIN

      The player has been automatically put on this node because it's a non-full main node.

    • TRANSFERRED

      Declaration

      TRANSFERRED

      The player has been transferred to this node.

    • FORCIBLY_TRANSFERRED

      Declaration

      FORCIBLY_TRANSFERRED

      The player was on a node that has wrongly disconnected (probably crashing) and the player has been transferred to the first non-full main node.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • reason

      Declaration

      ubyte reason;

      Reason for which the player has been added to the node.

    • type

      Declaration

      ubyte type;

      Game used by the player, which could either be Minecraft: Java Edition or Minecraft (Bedrock Engine). It should be one of the keys given in NodeInfo's acceptedGames field.

    • protocol

      Declaration

      uint protocol;

      Version of the protocol used by the client. Should be contained in the list given to the hub in the NodeInfo's acceptedGames field.

    • uuid

      Declaration

      UUID uuid;

      Client's UUID, given by Mojang's or Microsoft's services if the server is in online mode or given by the client (and not verified) if the server is in offline mode.

    • username

      Declaration

      string username;

      Username of the player.

    • displayName

      Declaration

      string displayName;

      Display name of the player, which can contain formatting codes. By default it's equals to the username but it can be updated by the node using the UpdateDisplayName packet.

    • gameName

      Declaration

      string gameName;

      Name of the game played by the client; for example Minecraft or Minecraft: Java Edition.

    • gameVersion

      Declaration

      string gameVersion;

      Version of the game used by the client, usually in the format major.minor[.patch], calculated by the server or given by the client during the authentication process. The node should verify that the version exists and matches the protocol indicated in the protocol field.

    • permissionLevel

      Declaration

      ubyte permissionLevel;

      Player's permission level that indicates its administration power. It's set to USER by default, which has no particular permission.

    • dimension

      Declaration

      ubyte dimension;

      Dimension in which the player was playing before being transferred in the MCPE format (0: overworld, 1: nether, 2: end). It shouldn't be considered if the client just joined the server instead of being transferred.

    • viewDistance

      Declaration

      uint viewDistance;

      Client's view distance (or chunk radius).

    • clientAddress

      Declaration

      Address clientAddress;

      Remote address of the player.

    • serverAddress

      Declaration

      ServerAddress serverAddress;

      Address used by the client to connect to the server. The value of this field is the address the client has saved in its servers list. For example a client that joins through localhost and a client that joins through 127.0.0.1 will connect to the same server with the same ip but the field of this value will be different (localhost for the first client and 127.0.0.1 for the latter).

    • skin

      Declaration

      Skin skin;

      Client's skin, given by the client or downloaded from Mojang's services in online mode.

    • language

      Declaration

      string language;

      Client's language, in the same format as HubInfo's language field, which should be updated from the node when the client changes it.

    • inputMode

      Declaration

      ubyte inputMode;

      Client's input mode. May be a controller, a mouse/keyboard set or a touch screen.

    • gameData

      Declaration

      JSONValue gameData;

      Example:

      1. // bedrock engine { "edu": false, "DeviceOS": 1, "DeviceModel": "ONEPLUS A0001" }

  • Remove

    Declaration

    struct Remove;

    Removes a player from the node. If the player is removed using Kick or Transfer this packet is not sent.

    • LEFT

      Declaration

      LEFT

      The player has closed the connection.

    • TIMED_OUT

      Declaration

      TIMED_OUT

      The hub has closed the connection because didn't had any response from the client for too long.

    • KICKED

      Declaration

      KICKED

      The player has been manually kicked.

    • TRANSFERRED

      Declaration

      TRANSFERRED

      The player has been transferred by the hub

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • reason

      Declaration

      ubyte reason;

      Reason of the disconnection.

  • Kick

    Declaration

    struct Kick;

    Kicks a player from the node and the whole server. When a player is disconnected from the node using this packet the hub will not send the Remove packet.

    • hubId

      Declaration

      uint hubId;

      Player to be kicked.

    • reason

      Declaration

      string reason;

      Reason of the disconnection that will be displayed in the client's disconnection screen.

    • translation

      Declaration

      bool translation;

      Whether the previous string should be translated client-side or not.

    • parameters

      Declaration

      string[] parameters;

      Optional parameters for the translation (Only for java clients).

  • Transfer

    Declaration

    struct Transfer;

    Transfers a player to another node. When a player is transferred from the node the hub will not send the Remove packet and there's no way, for the node, to know whether the player was disconnected or successfully transferred, if not using messages through a user-defined protocol.

    • DISCONNECT

      Declaration

      DISCONNECT

      Disconnect with End of Stream message.

    • AUTO

      Declaration

      AUTO

      Connect to the first available node or disconnects if there isn't one.

    • RECONNECT

      Declaration

      RECONNECT

      Connect to the same node, but as a new player.

    • hubId

      Declaration

      uint hubId;

      Player to be transferred.

    • node

      Declaration

      uint node;

      Id of the node that player will be transferred to. It should be an id of a connected node (which can be calculated using AddNode and RemoveNode packets), otherwise the player will be disconnected or moved to another node (see the following field).

    • onFail

      Declaration

      ubyte onFail;

      Indicates the action to be taken when a transfer fails because the indicated node is not connected anymore or it cannot accept the given player's game type or protocol. If the indicated node is full the player will be simply disconnected with the Server Full message.

  • UpdateDisplayName

    Declaration

    struct UpdateDisplayName;

    Updates the player's display name when it is changed. When this packet is sent by the node a copy is always sent back by the hub.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • displayName

      Declaration

      string displayName;

      Player's display name that can contain formatting codes. Prefixes and suffixes should be avoided.

  • UpdateWorld

    Declaration

    struct UpdateWorld;

    Updates the player's world. The player's dimension should be updated by the hub using worldId to identify the world added with AddWorld and removed with RemoveWorld.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • worldId

      Declaration

      uint worldId;

      World's id, that should have been previously added with the AddWorld packet.

  • UpdatePermissionLevel

    Declaration

    struct UpdatePermissionLevel;

    Update the player's permission level. When this packet is sent by the node a copy is always sent back by the hub.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

  • UpdateViewDistance

    Declaration

    struct UpdateViewDistance;

    Notifies the node that the player's view distance has been updated client-side. The node may decide to not accept the new view distance and not send the required chunks.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • viewDistance

      Declaration

      uint viewDistance;

      Player's new view distance as indicated by the client.

  • UpdateLanguage

    Declaration

    struct UpdateLanguage;

    Updates the player's language when the client changes it.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • language

      Declaration

      string language;

      Player's language in the same format as HubInfo's language field.

  • UpdateLatency

    Declaration

    struct UpdateLatency;

    Updates the latency between the player and the hub.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • latency

      Declaration

      uint latency;

      Player's latency in milliseconds. The latency between the client and the node is then calculated adding the latency between the node and the hub to this field's value.

  • UpdatePacketLoss

    Declaration

    struct UpdatePacketLoss;

    Updates the packet loss between the player and the hub.

    • hubId

      Declaration

      uint hubId;

      Player's unique id given by the hub.

    • packetLoss

      Declaration

      float packetLoss;

      Percentage of lost packets, from 0% (no packet lost) to 100% (every packet lost).
